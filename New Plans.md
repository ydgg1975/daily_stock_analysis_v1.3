# New Plans

daily_stock_analysis 주식분석 에이전트 고도화 작업의 우선순위 계획입니다.
구현이 완료된 항목은 아래 목록에서 지웁니다.

## 1. KR/US 기본 모드 명확화

- 제품 기본 지원 시장을 KR/US로 명시한다.
- CN/HK 백엔드 레거시 경로는 유지하되 기본 UI 노출에서는 숨긴다.
- 기본 모드에서 `600519` 같은 중국 A주 코드가 입력되면 지원 범위 안내를 표시한다.
- 설정/도움말/placeholder 문구에서 KR/US 기본 정책을 일관되게 맞춘다.

## 2. 한국 종목 입력 UX 보강

- 한국 종목 자동완성 데이터를 보강한다.
  - 삼성전자 -> `005930.KS`
  - SK하이닉스 -> `000660.KS`
  - KOSDAQ 대표 종목 -> `.KQ`
- bare 6자리 코드 입력 시 `.KS`/`.KQ` 명시 입력을 유도한다.
- 필요하면 후보 선택 UI를 추가한다.
- 백엔드는 명시 suffix `.KS`/`.KQ`를 우선 신뢰하도록 유지한다.

## 3. 한국 데이터 품질 보강

- pykrx 또는 별도 KRX provider 추가 여부를 결정한다.
- pykrx를 붙일 경우 가격, 시총, 투자자별 수급 중 1차 범위를 정한다.
- OpenDart 공시/재무 API 연동 여부를 결정한다.
- OpenDart를 붙일 경우 API 키 설정, `.env.example`, 문서를 함께 업데이트한다.
- 한국어 기업명/섹터 매핑 테이블을 보강한다.

## 4. AI 분석 언어와 출력 품질 고정

- 새 분석 요청에서 report language가 중국어로 돌아가지 않도록 고정한다.
- 한국 종목 분석 시 한국 시장 가이드라인을 명시적으로 주입한다.
- LLM 실패 fallback 문구가 중국어로 표시되지 않게 정리한다.
- 최종 목표 언어를 `ko`로 둘지 `en`으로 둘지 결정한다.
- `ko`를 목표로 하면 report language `ko` 지원 범위를 별도 구현한다.

## 5. 홈과 분석 기록 UI 정리

- 홈 화면 분석 작업 패널의 진행 문구와 상태 표시를 정리한다.
- 시장 선택 UI를 KR/US 중심으로 단순화한다.
- 포트폴리오 기본 통화와 시장 선택 흐름을 KRW/USD 기준으로 정리한다.

## 6. FastAPI 비동기 파이프라인 실제 연결

- `AsyncStockAnalysisPipeline`을 실제 API 엔드포인트에 연결할 범위를 정한다.
- 동기 파이프라인과 비동기 파이프라인의 응답 계약을 맞춘다.
- 작업 큐, 진행률 이벤트, 에러 처리 경로를 검증한다.
- API/Web/Desktop 호환 테스트를 추가한다.

## 7. 검색 캐싱과 조건부 AI 분석

- 뉴스 검색 결과 SQLite TTL 캐시 설계를 정한다.
- 동일 종목/동일 기간 분석 재사용 조건을 정한다.
- 횡보 종목 또는 변동성 낮은 종목의 고비용 AI 분석 생략 조건을 설계한다.
- 캐시 사용 여부와 생략 사유를 UI/API 응답에 표시한다.
