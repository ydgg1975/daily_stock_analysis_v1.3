# New Plans

daily_stock_analysis를 KR/US 중심의 한국어 주식 분석 앱으로 정리하기 위한 우선순위 작업 목록입니다.
구현이 완료된 항목은 이 문서에서 지워 나갑니다.

## 1. 한국어 리포트 모드 정착

- `REPORT_LANGUAGE=ko`가 기본값으로 동작하는지 백엔드와 설정 UI에서 검증한다.
- 새 분석 결과의 `reportLanguage`가 `ko`로 저장되는지 확인한다.
- 리포트 본문, dashboard, agent trace, fallback 문구가 중국어로 떨어지지 않는지 확인한다.
- 기존 `zh` legacy 기록은 새 분석 결과와 섞이지 않도록 삭제/숨김/경고 UX 중 하나로 정리한다.
- 한국어 리포트 smoke test를 추가한다.

## 2. Web 리포트 UI 한국어화 마무리

- 리포트 화면의 버튼, 카드 제목, gauge, trace, 상세 리포트 라벨을 한국어로 통일한다.
- 기존 `en` 또는 `zh` 기록을 열어도 앱 크롬 라벨은 한국어로 보이게 유지한다.
- 모바일/데스크톱 화면에서 긴 한국어 문구가 버튼이나 카드 밖으로 넘치지 않는지 확인한다.

## 3. 한국 주식 입력 UX 보강

- 삼성전자 -> `005930.KS`, SK하이닉스 -> `000660.KS` 자동완성 데이터를 보강한다.
- bare 6자리 코드 입력 시 `.KS`/`.KQ` 명시 입력을 유도한다.
- KOSPI/KOSDAQ 선택 또는 자동 추론 UX를 검토한다.
- 백엔드는 명시 suffix `.KS`/`.KQ`를 우선 신뢰하도록 유지한다.

## 4. 한국 시장 데이터 품질 보강

- pykrx 또는 별도 KRX provider 추가 여부를 결정한다.
- 가격, 시총, 외국인/기관 수급 중 1차 연동 범위를 정한다.
- OpenDart 공시/재무 API 연동 여부를 결정한다.
- OpenDart를 붙일 경우 API 키 설정, `.env.example`, 문서를 함께 업데이트한다.
- 한국어 기업명/티커 매핑 테이블을 보강한다.

## 5. 미국 시장 데이터 품질 보강

- Yahoo Finance 의존도를 낮추기 위한 보조 provider 후보를 정한다.
- Finnhub, AlphaVantage 등에서 우선 가져올 필드를 정한다.
- API 키가 없을 때도 분석 흐름이 중단되지 않는 fallback을 유지한다.

## 6. FastAPI 비동기 파이프라인 연결

- `AsyncStockAnalysisPipeline`을 실제 API 엔드포인트에 연결할 범위를 정한다.
- 동기/비동기 파이프라인의 응답 계약을 맞춘다.
- 작업 큐 진행률, 에러 처리, 취소/재시도 경로를 검증한다.
- API/Web/Desktop 호환 테스트를 추가한다.

## 7. 검색 캐싱과 조건부 AI 분석

- 뉴스 검색 결과 SQLite TTL 캐시 설계를 확정한다.
- 동일 종목/동일 기간 분석 재사용 조건을 정한다.
- 횡보 종목 또는 변동성 낮은 종목의 고비용 AI 분석 생략 조건을 설계한다.
- 캐시 사용 여부와 생략 사유를 API/UI 응답에 표시한다.
