### 1. 修复 `src/config.py` 中的环境变量加载
- **问题**：`load_dotenv` 默认 `override=False`。在 GitHub Actions 中，变量是通过 `env` 块显式传递的系统环境变量，`load_dotenv` 不会覆盖它们，这是正确的行为。但在本地开发时，若系统环境变量中存在空值（如 `GEMINI_API_KEY=`），会导致 `.env` 文件中的值被忽略。
- **修复**：
  - 修改 `setup_env` 函数，将默认值改为 `override=True`。**这对 GitHub Actions 没有负面影响**，因为 GitHub Actions 运行时通常不会有 `.env` 文件（即使有，其内容通常也应优先于系统默认值，但实际上 GitHub Actions 的 `secrets` 是通过系统环境变量传递的，`python-dotenv` 即使 override 也是覆盖它自己加载的内容，不会反向清除系统环境变量）。
  - **更稳妥的做法**：为了绝对安全，在 `_load_from_env` 方法中，**仅针对 `.env` 文件存在的情况**启用 override，或者在读取具体配置项时增加 `.strip()` 处理。
  - **关键动作**：在 `_load_from_env` 方法中，对 `gemini_api_key`、`tushare_token` 等敏感配置项增加 `.strip()` 处理。这能解决本地 `.env` 文件中可能存在的换行符或空格问题，同时兼容 GitHub Actions（Secrets 通常是纯文本，strip 无害）。

### 2. 增强 `src/analyzer.py` 中的 API Key 验证日志
- **问题**：当 Key 无效（如包含 `your_` 前缀或长度不足）时，`GeminiAnalyzer` 仅记录 "Gemini API Key not configured"，误导用户以为未读取到配置。
- **修复**：在 `__init__` 初始化逻辑中增加详细的验证失败日志，明确指出 Key 被拒绝的原因（如“包含占位符前缀”或“长度过短”）。
