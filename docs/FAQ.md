# FAQ

Daily Stock Analysis를 사용하면서 자주 만나는 문제와 해결 방법입니다.

## 데이터

### 미국 주식 가격이 이상하게 표시됩니다

미국 주식은 `AAPL`, `TSLA`처럼 영문 티커를 사용합니다. 한국/중국 숫자 코드와 충돌할 수 있는 경우에는 시장 접두어를 붙이는 방식을 권장합니다.

예:

```text
KR005930
CN600519
HK00700
AAPL
```

### 한국 종목이 검색되지 않습니다

먼저 종목 코드 형식을 확인하세요.

```text
KR005930
KR000660
```

한국 종목 인덱스나 데이터 공급자 설정이 최신인지 확인하고, Web UI 자동완성 결과가 비어 있다면 인덱스 재생성 또는 데이터 공급자 설정을 점검합니다.

### 데이터 공급자가 실패합니다

무료 데이터 공급자는 요청량 제한이나 일시 장애가 있을 수 있습니다. 시스템은 가능한 경우 다른 공급자로 fallback합니다.

권장 사항:

- 종목 수를 줄여 재시도
- 스케줄 간격 조정
- 데이터 공급자 우선순위 확인
- API Key가 필요한 공급자는 키 유효성 확인

## AI 모델

### 모델 호출이 실패합니다

다음을 확인하세요.

- API Key가 설정되어 있는지
- 모델명이 공급자에서 실제로 지원되는지
- Base URL이 올바른지
- 네트워크나 프록시 설정이 필요한지

Web 설정 화면에서 저장 후 테스트 호출을 먼저 실행하는 것이 좋습니다.

### 모델 응답이 비어 있습니다

가능한 원인:

- 모델 권한 부족
- 공급자 timeout
- 너무 긴 입력
- 모델 파라미터 미지원

다른 모델 또는 더 짧은 입력으로 재시도하세요.

## Web UI

### Web UI가 열리지 않습니다

프런트엔드 빌드가 없을 수 있습니다.

```bash
cd apps/dsa-web
npm ci
npm run build
```

개발 서버가 필요하면:

```bash
npm run dev
```

### 로그인 화면에서 막힙니다

초기 비밀번호 설정 상태와 `.env`의 인증 관련 설정을 확인하세요. 인증을 껐다 켜는 경우 기존 세션이 남아 있을 수 있으므로 브라우저 새로고침 또는 쿠키 삭제 후 다시 시도합니다.

## 알림

### 알림이 오지 않습니다

확인할 항목:

- 알림 채널별 URL/API Key
- 알림 라우팅 설정
- 최소 심각도 설정
- 조용한 시간대 설정
- Web 설정 화면의 알림 테스트 결과

단일 알림 채널 실패가 전체 분석을 중단시키지 않도록 설계되어 있으므로, 로그에서 어떤 채널이 실패했는지 확인하세요.

## Bot

### 봇 명령이 동작하지 않습니다

확인할 항목:

- 플랫폼별 Webhook 또는 Stream 설정
- 봇 토큰/API Key
- 명령 접두어
- 서버 외부 접근 가능 여부
- 봇이 대화방에서 멘션을 받아야 하는지 여부

기본 명령은 [Bot 명령](bot-command.md)을 참고하세요.

## 배포

### Docker에서 실행이 실패합니다

다음을 확인하세요.

- `.env`가 컨테이너에 전달되는지
- 볼륨 권한이 맞는지
- 로그 디렉터리에 쓰기 권한이 있는지
- 필요한 포트가 열려 있는지

### GitHub Actions가 실패합니다

주요 확인 항목:

- `python scripts/check_ai_assets.py`
- `python scripts/check_language_artifacts.py`
- Web 변경 시 `npm run lint`, `npm run build`
- 백엔드 변경 시 `./scripts/ci_gate.sh`

## 언어/문자 깨짐

### 중국어 또는 깨진 문자열이 다시 보입니다

다음 검사를 실행하세요.

```bash
python scripts/check_language_artifacts.py
python scripts/check_language_artifacts.py --pinyin-scope surface
```

사용자에게 보이는 문구는 한국어 또는 명확한 영어로 작성합니다.
