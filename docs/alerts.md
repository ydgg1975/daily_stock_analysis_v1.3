# 알림과 경고 정책

이 문서는 분석 결과에서 표시되는 경고와 운영 알림의 기본 원칙을 정리합니다.

## 경고 유형

| 유형 | 설명 |
| --- | --- |
| 데이터 경고 | 시세, 뉴스, 재무 데이터 수집 실패 |
| 분석 경고 | 모델 응답 실패 또는 리포트 생성 실패 |
| 시장 경고 | 급등락, 거래량 급증, 주요 이벤트 |
| 운영 경고 | 알림 채널 실패, 설정 누락, 스케줄 실패 |

## 표시 원칙

- 사용자가 바로 이해할 수 있도록 한국어로 표시합니다.
- 원인과 다음 조치를 함께 제공합니다.
- 단일 데이터 공급자 실패가 전체 분석 실패로 보이지 않게 구분합니다.
- 투자 판단 문구와 시스템 오류 문구를 섞지 않습니다.

## 예시

```text
뉴스 검색 결과가 부족합니다. 최신 뉴스 반영도가 낮을 수 있습니다.
```

```text
Telegram 알림 전송에 실패했습니다. Bot Token과 Chat ID를 확인하세요.
```

## 점검 항목

- Web UI, API 응답, 알림 메시지의 표현이 같은지
- 깨진 문자나 중국어 원문이 남아 있지 않은지
- 경고가 너무 길어 모바일 알림에서 잘리지 않는지
- 실패 원인이 로그에 충분히 남는지
